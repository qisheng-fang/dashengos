## 前端无法对话 — 根因: http_proxy 环境变量 (10:43-10:55)

### 问题
前端对话功能无法使用。POST 请求到后端全部超时 (curl exit code 28)。

### 根因分析
之前误判为 `tsx watch` 导致 Fastify POST 请求 hang。**真正根因是系统 http_proxy 环境变量** (`http://127.0.0.1:53840`)：
- curl POST 请求走了 http_proxy 代理，代理转发 POST 请求有问题
- GET 请求受影响较小（代理可以转发）
- `no_proxy` 环境变量已包含 `127.*`，但 curl 对 `no_proxy` 通配符处理不一致
- `curl --noproxy '*'` 绕过代理后 POST 完美工作

### 修复
1. **Vite proxy 配置** (`apps/web/vite.config.ts`):

## 工作台主界面 ChatCopilot 消息不渲染 — 最终修复（第四轮 14:05-14:10）

### 问题
老板截图反馈：左侧会话列表有会话标题，但主区域仍显示「工作台仪表盘」，底部显示「0 条消息」。

### 排查过程
1. **agent-browser 验证**: 发送消息后 localStorage 有 3 条消息（欢迎语 + 用户 + AI 回复）✅
2. **但 UI snapshot 显示**: 只有欢迎语在 DOM 里，用户消息和 AI 回复都没有渲染
3. **🎯 根因定位**: `ChatCopilot.tsx` 第 101 行：
   ```js
   const { chatHistory, ... } = useAppStore()  // chatHistory 是 getter，不触发 React 重渲染！
   ```
   Zustand 的 getter（计算属性）解构后 **不会触发 React 组件重渲染**。`chatHistory` 定义为：
   ```js
   get chatHistory() {
     const state = get()
     const conv = state.conversations.find(c => c.id === state.activeConversationId)
     return conv?.messages ?? []
   }
   ```
   React 组件解构时只拿到 getter 的 **快照值**，store 更新后不会自动重渲染。

### 修复
**`ChatCopilot.tsx`** 改动：
```js
// 之前（不触发重渲染）
const { chatHistory, ... } = useAppStore()

// 之后（手动计算，触发重渲染）
const { conversations, activeConversationId, ... } = useAppStore()
const currentConv = conversations.find(c => c.id === activeConversationId)
const chatHistory = currentConv?.messages ?? []  // 手动计算，依赖 conversations/activeConversationId
```
不再从 store 解构 `chatHistory` getter，改为手动从 `conversations` 数组计算。
