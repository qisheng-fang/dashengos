### 问题

### 修复
- `chat.ts` 流式路径: 首次发送 `status: 等待模型响应...`
- **核心修复**: 把状态注入从 chat.ts 移到 `openAIStream()` 内部 (streaming.ts)

  - 状态文案从 `getStatusText()` 动态获取，经 DB 缓存+硬编码回退
  - 6 阶段: 领导潜台词解码 → AI思考 → 深度分析 → 生成内容
- 非流式回退路径: 同样增加按字符数阈值(50/200/600)发送 status 事件
- 前端已正确解析 `event: status` → `setStream.statusText` → 状态栏左侧显示

### 效果
用户发送消息后，状态栏从左到右实时显示:
1. 🌐 等待模型响应... → 💭 AI 思考中 → 🧠 正在深度分析 → ✨ 正在生成内容

## Harness 框架 + 超级 System Prompt (19:10-19:50)

### 核心: DaShengOS 聊天降智诊断 → Harness 框架建设

**降智根因 (4 层)**:
1. LLM_PROVIDER=agnes_ai → agnes-2.0-flash 是轻量 flash 模型，非推理型
2. System Prompt 极度简陋 (仅 1 句话)
3. 完全缺失 Harness 框架 (记忆/规划/反思/验证)
4. DeerFlow 竞速逻辑偏向短答

**解决方案 — 5 个 Harness 模块**:

1. **system-prompt.ts** — 超级 System Prompt
   - 品牌知识注入 (爱尤趣行业/受众/卖点/竞品/渠道)
   - 角色定位 (战略师+内容生产+分析师)
   - 能力边界 (16 工具 + 10 Agent)
   - 输出规范 (摘要→分析→行动项→风险)
   - 第一性原理思维引导
   - 反思协议 (自检 4 项)
   - 动态任务模式增强 (marketing/analysis/technical/creative)

2. **memory.ts** — 记忆注入
   - 品牌知识硬编码 (始终注入)
   - 用户学习记录 (learnings 表)
   - 用户记忆 (memory 表)
   - 最近对话话题提取

3. **planner.ts** — 规划分解
   - 任务复杂度评估 (simple/moderate/complex)
   - LLM 辅助规划 (first principles 分解)
   - 降级回退 (LLM 不可用时静态规划)

4. **reflector.ts** — 反思验证
   - 幻觉检测 (数据点无"约/估算"限定词)
   - 空结果/截断/偏题检测
   - 自动重试策略 (rephrase/different_tool/escalate)
   - 反思 prompt 生成

5. **index.ts** — 编排器
   - Stream 模式: 轻量增强 (prompt+记忆)
   - Agent 模式: 完整编排 (记忆+规划+执行+反思+验证)
   - 工具序列推荐

**接入点**:
- `/api/v1/chat/stream` — SSE 流式走 enhanceStreamMode() 注入 Harness prompt
- `/api/v1/chat/agent` — Agent Loop 走 prepareAgentMode() 完整 Harness + systemPrompt 传入
- `directLLM()` — REST 端点也走 Harness 增强 prompt
- `LLM_PROVIDER` **已从 agnes_ai 改为 siliconflow** (Qwen2.5-72B)

**文件**:
- `packages/backend/src/core/harness/{index,system-prompt,memory,planner,reflector}.ts`
- `packages/backend/src/api/chat.ts` (v4.0 Harness 增强版)
- `packages/backend/.env` (LLM_PROVIDER=siliconflow)

## 工作台主界面 ChatCopilot 消息不渲染 — 最终修复（第四轮 14:05-14:10）

### 问题
老板截图反馈：左侧会话列表有会话标题，但主区域仍显示「工作台仪表盘」，底部显示「0 条消息」。

1. **agent-browser 验证**: 发送消息后 localStorage 有 3 条消息（欢迎语 + 用户 + AI 回复）✅
2. **但 UI snapshot 显示**: 只有欢迎语在 DOM 里，用户消息和 AI 回复都没有渲染
3. **🎯 根因定位**: `ChatCopilot.tsx` 第 101 行：
   ```js
   const { chatHistory, ... } = useAppStore()  // chatHistory 是 getter，不触发 React 重渲染！
   ```
   Zustand 的 getter（计算属性）解构后 **不会触发 React 组件重渲染**。`chatHistory` 定义为：
   ```js
   get chatHistory() {
     const state = get()
     const conv = state.conversations.find(c => c.id === state.activeConversationId)
     return conv?.messages ?? []
   }
   ```
   React 组件解构时只拿到 getter 的 **快照值**，store 更新后不会自动重渲染。

### 修复
**`ChatCopilot.tsx`** 改动：
```js
// 之前（不触发重渲染）
const { chatHistory, ... } = useAppStore()

// 之后（手动计算，触发重渲染）
const { conversations, activeConversationId, ... } = useAppStore()
const currentConv = conversations.find(c => c.id === activeConversationId)
const chatHistory = currentConv?.messages ?? []  // 手动计算，依赖 conversations/activeConversationId
```
不再从 store 解构 `chatHistory` getter，改为手动从 `conversations` 数组计算。
