## 23:07 对齐 Hermes 设计哲学（老板痛点 2）

**老板原话**："hermes 能自动配置模型、WebUI 前端只有入口、后端全管、模块状态都在后端、前端只看能不能完成。缺技能也能全面分析提示"

**翻译成 3 条设计原则**：
1. **后端厚、前端薄、状态真源在 `~/.hermes/`** — web_server 4442 行 / 69 API / config.yaml + .env + auth.json 三层
2. **新功能 = 丢一个目录即可** — 16+ provider 各在 `plugins/model-providers/<name>/`，pkgutil.iter_modules 自动扫描
